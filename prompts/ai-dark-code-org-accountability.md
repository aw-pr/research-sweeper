# Research Brief: AI Dark Code — Organisational Accountability and Control

## Topic string (paste into --topic)

```
AI-generated and agent-produced code ("dark code") in enterprise settings June 2025–April 2026: organisational accountability structures, failure and adaptation of established management frameworks, technical and governance controls, observability and discoverability of agent logic, and documented outcomes from early enterprise adoption.
```

## Sub-questions for synthesis to address

**Management theory under strain**
- Which classical management frameworks — principal-agent theory, stewardship theory, command-and-control, OKRs, RACI — break down when production logic has no human author, and precisely where does the failure occur?
- Which management approaches have shown unexpected resilience or applicability — e.g. risk-based models from financial services, safety-case methods from engineering, or model risk management from algorithmic trading?
- Where teams have applied traditional change management (ITIL, CAB processes) to agent-generated artefacts, what adaptations were required and what was abandoned?
- Is there emerging consensus in management research on a new theoretical frame, or is the field still borrowing analogies?

**Organisational accountability and structure**
- How are enterprises restructuring roles, ownership, and accountability chains when production logic is generated by agents rather than authored by engineers?
- What governance gaps have emerged where dark code sits outside traditional code review, audit, or change management processes?
- Have any organisations reversed or paused AI code generation initiatives — what were the stated triggers (incident, compliance, audit failure)?
- How are accountability disputes being resolved when an agent-produced artefact causes a production incident — who owns it?

**Technical controls and governance frameworks**
- What technical controls (policy-as-code, automated lineage tracking, sandboxing, signed artefacts, prompt audit logs) have been deployed specifically for agent-generated outputs?
- How are enterprises handling versioning, rollback, and incident attribution for code with no human author?
- What standards bodies or regulatory frameworks are beginning to address dark code provenance and auditability?
- Where have control frameworks been tried and failed — what patterns led to failure?

**Observability, telemetry, and discoverability**
- What tooling exists for making agent-generated logic inspectable at runtime — tracing, lineage graphs, structured prompt logs, OpenTelemetry extensions for LLMs?
- How is institutional knowledge about what agent-produced code does being captured and made discoverable across teams — wikis, embeddings, automated documentation?
- Are there emerging standards for agent telemetry, and which enterprises are early adopters?
- What is the state of static analysis and explainability for agentic outputs vs. human-authored code?
